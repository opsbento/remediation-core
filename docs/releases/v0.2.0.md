# remediation-core v0.2.0

## Changes

- Supports multiple direct dependency updates in one remediation run.
- Prioritizes remediation candidates by highest vulnerability severity before dependency name.
- Adds `dependencies[]` to `result.json` while keeping the existing `dependency` field for backward compatibility.

## Recommended usage

Use this release from `security-workflows` with `maximum-updates` greater than `1` when batching dependency fixes is desired.

## Compatibility

`dependency` remains populated with the first selected update. Consumers should prefer `dependencies[]` when present.
