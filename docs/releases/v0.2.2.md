# remediation-core v0.2.2

## Changes

- Adds a general parent-remediation extension point for transitive findings.
- Implements npm direct-parent detection from `package-lock.json`.
- Allows transitive npm findings to be remediated by updating the owning direct dependency, then verifying the original transitive findings are removed.

## Compatibility

Use this release for production batching when scan failures include transitive npm packages such as dependencies introduced by `express`.
