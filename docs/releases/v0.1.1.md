# remediation-core v0.1.1

## Changes

- Publishes Linux CLI release assets for `remediate-linux-amd64` and `remediate-linux-arm64`.
- Publishes `checksums.txt` for release asset verification.
- Sets explicit Syft directory source metadata to suppress noisy scan warnings in GitHub Actions.

## Recommended usage

Use this release from `security-workflows` through the pinned CLI release settings:

- `remediation-core-repository`: `opsbento/remediation-core`
- `remediation-core-version`: `v0.1.1`
- `remediation-core-asset`: `remediate-linux-amd64`

## Verification

The release workflow runs `go test ./...`, builds Linux CLI assets, and generates SHA-256 checksums before creating or updating the GitHub Release.
