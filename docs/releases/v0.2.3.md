# remediation-core v0.2.3

## Changes

- Adds `verification.remaining_threshold_findings` to `result.json`.
- Marks remediation as `FAILED` when findings at or above the configured minimum severity still remain after the selected updates.
- Adds an explicit message for incomplete remediation so workflows do not create merge-ready PRs that would still fail the scan gate after merge.

## Compatibility

Use this release when caller repositories require remediation PRs to represent a clean threshold scan result, not only removed target findings.
