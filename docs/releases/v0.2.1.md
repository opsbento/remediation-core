# remediation-core v0.2.1

## Changes

- Verifies each candidate update before accepting it.
- Rejects candidate versions that introduce findings at or above the configured minimum severity.
- Continues to the next available candidate version when a lower candidate is not clean enough.

## Compatibility

Use this release instead of `v0.2.0` for production batching. `v0.2.0` introduced multi-update support, but `v0.2.1` tightens candidate safety.
